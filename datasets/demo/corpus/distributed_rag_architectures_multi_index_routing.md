Distributed RAG Architectures — это системы, где запросы маршрутизируются (routing) к разным индексам в зависимости от типа вопроса, языка или предметной области для повышения качества и масштабирования.

Ключевые термины: Query Router (маршрутизатор запросов), Multi-Index Architecture (многокомпонентная архитектура), Federation (федерация), Content Router (контентный роутер).

Главные тезисы:

Router анализирует запрос и определяет, к какому индексу обратиться (например, техническая документация vs HR-политики).
Позволяет использовать специализированные индексы (разные модели эмбеддингов, разные чанк-стратегии) для разных типов контента.
Router может быть rule-based (по ключевым словам) или ML-based (классификатор запросов).
