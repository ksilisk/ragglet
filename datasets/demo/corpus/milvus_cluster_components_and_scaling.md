Milvus Cluster — распределенная архитектура для горизонтального масштабирования. Состоит из нескольких независимых компонентов, каждый из которых масштабируется отдельно.

Ключевые термины: Coordinator (координатор), Data Node (узел данных), Query Node (узел запросов), Index Node (узел индексации), Proxy (прокси), Message Storage (хранилище сообщений).

Главные тезисы:

Архитектура разделяет обязанности: Data Node пишет данные, Query Node ищет, Index Node строит индексы.
Хранилище сообщений (Pulsar/Kafka) гарантирует надежность и воспроизводимость операций.
Каждый компонент масштабируется независимо: при росте нагрузки на поиск добавляются Query Node, при росте объема данных — Data Node.
