Настройка производительности Milvus включает оптимизацию индексов, конфигурации компонентов и управление ресурсами для достижения целевых показателей latency и throughput.

Ключевые термины: Segment Size (размер сегмента), Index Building (построение индекса), Resource Group (группа ресурсов), Query Node Cache (кэш узла запросов).

Главные тезисы:

Ключевые параметры: nprobe (IVF), ef_search (HNSW), размер сегмента (рекомендуется 512 MB), настройки кэширования.
Troubleshooting: медленные запросы часто связаны с большими сегментами без индекса или малым nprobe; OOM — с недостаточным лимитом памяти для HNSW.
Используйте milvus_cli для диагностики состояния сегментов и индексов.
