Кэширование (Caching) в RAG — критический компонент для снижения latency и разгрузки дорогостоящих операций (генерация эмбеддингов, LLM-запросы) с помощью Embedding Cache и Retrieval Cache.

Ключевые термины: Embedding Cache (кэш эмбеддингов), Retrieval Cache (кэш результатов поиска), LLM Cache (кэш ответов), TTL (Time To Live), LRU (Least Recently Used).

Главные тезисы:

Embedding Cache хранит векторы для часто встречающихся запросов, экономя вызовы к моделям эмбеддингов.
Retrieval Cache хранит результаты поиска (найденные чанки) для повторяющихся запросов.
Кэширование ответов LLM (prompt → ответ) дает максимальный выигрыш, но требует осторожности с чувствительными данными.
