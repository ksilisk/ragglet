Sharding (шардирование) и Replication (репликация) в Qdrant — механизмы распределения данных для горизонтального масштабирования и отказоустойчивости. Routing определяет, как запросы направляются к данным.

Ключевые термины: Shard (шард), Replica (реплика), Distributed Deployment (распределенное развертывание), Consensus (консенсус), Raft.

Главные тезисы:

Sharding распределяет точки по разным узлам по хешу ID — каждый шард содержит только часть данных.
Replication создает копии шардов на разных узлах для защиты от сбоев. Qdrant использует Raft для консенсуса между репликами.
Routing: при запросе координатор (peer) определяет, какие шарды содержат нужные данные, и отправляет запрос параллельно на все реплики (или на одну, в зависимости от настроек).
