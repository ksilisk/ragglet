PostgreSQL предлагает стандартные механизмы масштабирования: партиционирование (partitioning) для больших таблиц и репликацию (replication) для отказоустойчивости и распределения чтения.

Ключевые термины: Partitioning (партиционирование), Table Inheritance (наследование таблиц), Streaming Replication (потоковая репликация), Logical Replication (логическая репликация), Read Replicas (реплики для чтения).

Главные тезисы:

Партиционирование разделяет большую таблицу на физические куски по ключу (например, дате), ускоряя запросы с фильтрацией по этому ключу.
Streaming Replication создает физические копии (standby) для отказоустойчивости и балансировки чтения.
Logical Replication позволяет реплицировать только определенные таблицы и поддерживает разные версии PostgreSQL.
