RAG (Retrieval-Augmented Generation) — это архитектурный паттерн, который дополняет знания LLM актуальными данными из внешних источников перед генерацией ответа.

Ключевые термины: Retriever (поисковик), Generator (генератор), Index (индекс), Embeddings (эмбеддинги), Context (контекст).

Главные тезисы:

Основная цель RAG — повысить точность и актуальность ответов, уменьшая «галлюцинации».
Пайплайн состоит из трех этапов: индексация знаний, поиск релевантных документов по запросу и генерация ответа с их учетом.
Качество финального ответа критически зависит от качества найденного контекста.
RAG позволяет LLM обращаться к данным, которые не входили в её обучающую выборку (базы знаний, документы компании).
